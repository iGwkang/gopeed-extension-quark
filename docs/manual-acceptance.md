# 手工验收清单

以下项目需要使用测试账号的真实夸克 Cookie 和可控分享链接，由用户在 Gopeed 开发者模式中执行。请勿把 Cookie、提取码或含敏感参数的完整日志提交到仓库。

## 准备

- [ ] 执行 `npm install && npm run build`。
- [ ] 在 Gopeed **扩展**页连续点击 **安装** 5 次，开启开发者模式并选择本项目根目录。
- [ ] 从浏览器网盘文件列表请求中复制包含 `__puus=` 的完整 Cookie，填入扩展设置。
- [ ] 准备无提取码分享、带提取码分享和至少含一层子目录的分享。
- [ ] 记录测试前网盘可用空间及 `GopeedTemp` 内容。

## 验收项目

### 1. 无提取码分享与目录结构

- [ ] 用无提取码分享链接创建任务，可以递归列出全部文件。
- [ ] 下载后，本地子目录层级与分享中的相对目录一致，文件未被拍平。

### 2. 提取码

- [ ] 使用 `https://pan.quark.cn/s/{shareId}?pwd={正确提取码}`，可以正常解析。
- [ ] 将 `pwd` 改为错误值，界面显示明确的中文提取码错误。
- [ ] 删除需要提取码链接中的 `?pwd=`，界面提示在链接中补充提取码。

### 3. 默认清理

- [ ] 保持“解析完毕后释放网盘空间”为开启状态并解析分享。
- [ ] 获取直链后，本次转存文件已从 `GopeedTemp` 删除。
- [ ] `GopeedTemp` 文件夹本身仍保留，其他原有文件未被删除。

### 4. 保留转存与直链刷新

- [ ] 关闭“解析完毕后释放网盘空间”并解析分享，转存文件保留在 `GopeedTemp`。
- [ ] 在测试环境中让下载直链失效或替换为无效旧链接，再启动任务。
- [ ] `onStart` 使用保存的 `fid` 获取新直链，任务可继续下载且无需重复保留额外副本。

### 5. Cookie 错误

- [ ] 清空 Cookie 后解析，界面显示如何配置 Cookie 的中文错误。
- [ ] 填入不含 `__puus` 的 Cookie 后解析，界面显示需包含 `__puus` 的中文错误。
